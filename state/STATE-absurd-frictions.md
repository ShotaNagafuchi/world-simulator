# 方法論

イノベーションの種を見つけるには、摩擦を抽象的に（「コストが高い」）ではなく、
**人間が実際にやっている動作を文字通り描写**して、馬鹿馬鹿しいかどうかで判定する。

過去のイノベーションは全て「描写したら馬鹿げていた動作」を消している：
- 「金属の箱にファイルを入れて電車で運んでいる」→ GigaFile/Dropbox
- 「知りたいことを単語に分解してテキストボックスに打っている」→ ChatGPT
- 「道に立って腕を振って車を止めようとしている」→ Uber

## フィルター条件

1. **全員がやっている**（特殊な人だけではない）
2. **因果が即時**（うまくいった/いかないが秒で分かる）
3. **描写したら馬鹿げている**（「なんでこんなことやってるの？」）

# PayPalモデル適合性分析

## カテゴリ別の構造

### 「自分の情報をゼロから再入力する」（#1,6,7,8,9,10）

**共通する馬鹿げた構造**: あなたの情報は既にデジタルで存在しているのに、毎回ゼロから入力/説明させられる。

**全く別の方法**: あなたの「デジタルプロフィール」が、あなたの許可でサービスに自動的に渡される。

| PayPal条件 | 評価 |
|---|---|
| 必須 | ◎ 全員が毎週やっている |
| 因果即時 | ◎ 入力が不要になった/ならないは即座に分かる |
| 両面 | ◎ 利用者 × サービス提供者 |
| 低信頼開始 | ◎ まず名前と住所だけから始められる |
| 能力拡張 | ◎ 「どのサービスでも自分を知ってくれている」は新能力 |
| **計** | **5.0** |
| 窓 | △ Apple/Googleが部分的にやっている（自動入力）。しかし**サービス間の文脈共有**はできていない |

**これはPayPalモデルのフル適合。** 問題は、Apple/Googleのオートフィルで「入力」は部分的に解決済み。しかし「文脈の共有」（前回の通話内容、病歴、好み）は未解決。

**ロックイン構造**: あなたの全文脈データがこのプラットフォームに蓄積される → 他に移ると全サービスが「あなたを知らない」状態に戻る → 抜けられない。これはAppleのiCloudロックインに似ているが、対象が「デバイスデータ」ではなく「あなたという人間の全文脈」。

# 発見

## 「文脈の分断」と「情報の分断」が最大の馬鹿げた摩擦

2つに共通する根本問題: **「あなた」に関する情報が、あなたの周りに散らばっていて、どのサービスも全体像を持っていない。**

- 銀行Aはあなたの口座残高を知っている。銀行Bも知っている。でも合計は誰も知らない
- 病院Aはあなたの手術歴を知っている。病院Bは知らない
- カスタマーサポートは前回の通話内容を知らない
- 新しい美容師はあなたの好みを知らない
- 各サービスの「あなた」は断片でしかない

**「あなた」の全体像を持つプレイヤーが存在しない** = ルーティングの空白（LAW-009）。

全く別の方法: **あなたの全文脈を1箇所に集約し、あなたの許可で各サービスに渡す「パーソナルコンテキストハブ」**。

これは:
- PayPalが「メールアドレスだけで送金」を可能にしたように、「名前を言うだけで全文脈が伝わる」を可能にする
- 因果即時: 情報が渡った/渡らないは即座に分かる
- 両面: 利用者 × サービス提供者
- ロックイン: 文脈データの蓄積が最強のスイッチングコスト（全サービスが「あなたを忘れる」）
- 能力拡張: 「どこに行っても自分を知ってくれている」は新能力

### ただしCON-002の壁

これだけの個人データを預けるには、責任の所在が必要（CON-002）。
Apple/Googleが最も近い位置にいる（既にデバイスデータを持っている）。
独立プレイヤーの窓は狭いが、**Apple/Googleは「デバイスのデータ」を持っているだけで「あなたの文脈」は持っていない**。

医療記録、保険の全体像、カスタマーサポートの履歴、好み——これらはAppleにもGoogleにもない。
