# 根源的欲求の市場マップ

CON-007の10の根源的欲求を、市場規模・既存の解決手段・イノベーション余地で分析する。

## PayPalモデルの条件（フィルター）

PayPal型イノベーション = 以下の5条件をすべて満たす：

1. **必須の行動**: 全員がやらざるを得ない（オプショナルではない）
2. **因果が明確**: 結果が即座に分かる（CON-004: フィードバックが速い）
3. **ルーティング可能**: A×Bの両面構造がある（LAW-009）
4. **低信頼で開始可能**: 小さい賭けから始められる（CON-004）
5. **能力拡張**: 判断の代替ではなく新しい能力（CON-008）

## 欲求別分析

### 1. 生存（食べたい、健康でいたい、長生きしたい）

市場規模: ~$20T（ヘルスケア$9T + 食品$9T + 住居$3.5T）
既存の解決手段: 病院、薬局、スーパー、不動産会社
選択肢の多さ: ★★★★★（膨大）
取得コスト（CON-014）: 食品は低い、医療は高い

PayPalモデル適合:
- 必須の行動: ◎（全員が食べ、医療を受ける）
- 因果が明確: △（食品は明確、健康は長期＝CON-004失敗）
- ルーティング: △（患者×医師のマッチングは既にある）
- 低信頼で開始: ✕（医療はCON-002の壁が極めて高い）
- **判定: ✕ 医療は制度制約が重すぎる。食品は選択肢が多すぎて差別化困難**

### 2. 安全（脅威から守られたい、失いたくない）

市場規模: ~$8T（保険$6T + セキュリティ$350B + 法務$900B）
既存の解決手段: 保険会社、警備会社、法律事務所
選択肢の多さ: ★★★（中程度、規制で寡占）
取得コスト: 高い（保険の比較は苦痛、法律相談は$300+/h）

PayPalモデル適合:
- 必須の行動: ◎（全員がリスクを持つ）
- 因果が明確: △（保険は事故まで分からない、法務は結果が遅い）
- ルーティング: ○（被保険者×保険会社のマッチングに余地）
- 低信頼で開始: △（金額が大きい＝信頼が前提）
- **判定: △ 保険比較は取得コスト削減の余地があるが、因果証明が遅い**

### 3. つながり（相手と繋がりたい、愛されたい）

市場規模: ~$2T（通信$1.7T + SNS$250B + マッチング$10B）
既存の解決手段: LINE, WhatsApp, Meta, マッチングアプリ
選択肢の多さ: ★★★★（多い、ネットワーク効果で寡占化済み）
取得コスト: 低い（アプリは無料）

PayPalモデル適合:
- 必須の行動: ◎（全員が繋がりを求める）
- 因果が明確: ◎（メッセージ送受信は即時確認）
- ルーティング: ◎（人×人の両面ネットワーク）
- 低信頼で開始: ◎（無料で始められる）
- **判定: ◎ だが既にLINE/WhatsApp/Metaが独占。窓が開いていない**

### 4. 所属（集団に属したい、仲間外れにされたくない）

市場規模: ~$500B（コミュニティ、ファンクラブ、宗教団体、同窓会、DAO）
既存の解決手段: SNS、Discord、Slack、宗教組織

- **判定: △ コミュニティは分散しており、ルーティングの統一が難しい**

### 5. 地位（認められたい、他者より上に立ちたい）

PayPalモデル適合:

- **判定: △ 地位は測定困難（CON-004失敗）。既存手段が多すぎる**

### 6. 自律（自分で決めたい、自由でいたい）

### 7. 互恵/贈与（恩を返したい、贈りたい、公正に扱われたい）

PayPalモデル適合:

- ルーティング: ○（贈り手×受け手の両面構造）

# 発見

**PayPalモデルが成立する条件を満たし、かつ窓が開いている欲求はほぼない。**

- 高スコアの欲求（つながり、好奇心）は既に独占者がいる
- 窓が開いている欲求（互恵/贈与、安全）はスコアが中程度
- 快楽は選択肢が多すぎて差別化不能
- 意味は因果証明が不可能

これは**PayPalモデル（ルーティング×即時フィードバック×両面ネットワーク）が「革命的」だった理由**を裏付ける。条件が揃う組み合わせは極めて稀。
