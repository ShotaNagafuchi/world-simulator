# 分析の前提

抽象的な「欲求」ではなく、人間が実際にやる**具体的な行動**で分析。PayPalは「遠くの人に金を送る」、Googleは「知りたいことを見つける」、Uberは「今すぐ車に乗る」。行動の粒度はこのレベル。

## PayPalモデル5条件（各1点）

1. **必須**: 全員がやらざるを得ない（高頻度）
2. **因果即時**: 結果が即座に分かる
3. **両面**: A×Bの両面ルーティング構造
4. **低信頼**: 小さく始められる
5. **能力拡張**: 今まで不可能だったことが可能に

## 凡例

- 摩擦: 低/中/高/極高
- 窓: ◎開/○半開/△狭い/✕閉
- AI: ◎桁で改善/○大幅改善/△部分改善/✕困難

# パターン分析

## 上位に共通する構造

1. **「専門家に高額を払うか、知らないまま損するかの二択」の破壊**
   - 契約書（弁護士$300/h → AI無料）
   - 保険比較（FP$200/h → AI無料）
   - 見積もり適正価格（コンサル → AI無料）
   - 労働権利（弁護士 → AI無料）

2. **「全員がやるのに誰も正しくやれていない」行動**
   - 契約書に署名（読まずにサイン）
   - 保険に加入（営業マンの言いなり）
   - 利用規約に同意（読まずにクリック）
   - 確定申告（間違っているかも分からない）

3. **CON-014（取得コスト支配）の逆転**
   - 今: 正しい情報の取得コスト > 間違った情報の取得コスト → 間違った方を選ぶ
   - AI後: 正しい情報の取得コスト ≈ 0 → 正しい方を選べる
