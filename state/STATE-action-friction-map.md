# 分析の前提

抽象的な「欲求」ではなく、人間が実際にやる**具体的な行動**のレベルで分析する。PayPalは「つながりの欲求」を解決したのではなく「遠くの人に金を送る」という行動の摩擦をゼロにした。

## PayPalモデル5条件（各1点、計5点）

1. **必須**: 全員がやらざるを得ない（頻度が高い）
2. **因果即時**: 結果が即座に分かる（CON-004）
3. **両面ルーティング**: A×Bの構造がある（LAW-009）
4. **低信頼開始**: 小さく始められる（CON-004 + CON-006）
5. **能力拡張**: 今まで不可能だったことが可能になる（CON-008）

追加評価軸:
- **現在の摩擦**: 今どれだけ面倒か（高いほどチャンス）
- **既存独占者**: 窓が開いているか
- **AIで10倍か**: AIの摩擦削減が桁か

# 発見

## 最有力3行動

### 1. リアルタイム異言語会話（4.5点、窓開、AI◎）

- **必須**: 全世界で言語の壁がある。ビジネス、旅行、移住、国際結婚
- **因果即時**: 通じた/通じないは即座に分かる
- **両面ルーティング**: 話者A × 話者B。両者がプラットフォームを使う必要がある
- **能力拡張**: 今まで不可能だった「母語だけで世界中の人と対話」が可能に
- **窓**: DeepLは文書翻訳、Google翻訳はカメラ翻訳。**リアルタイム音声会話の決定的プレイヤーがいない**
- **ロックイン構造**: 会話履歴＋学習データ蓄積 → 使うほど自分の話し方に最適化 → 抜けられない

### 2. 契約書/規約の即座の不利判定（4.5点、窓開、AI◎）

- **必須**: 全員が契約書にサインする（賃貸、SaaS、保険、雇用）
- **因果即時**: 「この条項は不利」の判定は即座に分かる
- **能力拡張**: 今まで弁護士$300/hでしかできなかったことが即座に可能に
- **窓**: **誰もやっていない**
- **CON-002の壁**: ただし「法的助言」ではなく「不利条項のハイライト」ならCON-002を迂回（判断代替ではなく能力拡張）
- **ロックイン構造**: 契約書の蓄積 → 自分の契約ポートフォリオ管理 → 抜けられない

### 3. 保険/金融商品の公正比較（4.0点、窓開、AI◎）

- **必須**: 全員が保険に入り、金融商品を選ぶ
- **因果**: 中期だが「A社よりB社が年間X万円安い」は即座に分かる
- **両面ルーティング**: 消費者 × 保険会社のルーティング
- **窓**: 既存比較サイトはLAW-008で腐っている。**買い手側のAIプレイヤーがいない**
- **CON-014**: 今は「営業マンが来るから話を聞く」（取得コストゼロの情報が勝つ）。AIが取得コストゼロで公正な比較を出せば、構造が変わる

## パターン

3つに共通するのは：**「今は専門家に高額を払うか、諦めるかの二択」の領域で、AIが取得コストをゼロにする**。

これは CON-014（取得コスト支配）× CON-006（10倍便利なら信用を超える）の組み合わせ。「弁護士$300/h vs AIが無料で即座」は10倍どころではない差。
