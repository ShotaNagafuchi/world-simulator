シミュレーション（simulations/）は世界を記述する。ポジションは**その世界の中で自分が張る賭け**である。
7つの質問（ティール）すべてに答えられないポジションは登録できない——validate が機械的に強制する。
各答えは感想ではなく、このリポジトリのオブジェクト（法則・制約・事例・観測・予測）を引用して書く。

# 1. エンジニアリング — ブレークスルーか、段階的改善か

リポジトリ語彙では：**どの定数を変数化するか**（constraints/ のどれを解凍するか）。
段階的改善 = 既に変数化された定数の上での最適化（つるはし。LAW-006 の寿命に従う）。
ブレークスルー = frozen/thawing の制約を自分で解凍する（CASE-2015-spacex-reusability 型）。

# 2. タイミング — なぜ今か

リポジトリ語彙では：**どの層がいま解凍中か**。state/ の決定的観測と history/ のコスト曲線で答える。
「窓」を定義する：何が観測されたら早すぎ／遅すぎと判定するか。

# 3. 独占 — 大きなシェアを取れる小さな市場から始めているか

リポジトリ語彙では：ボトルネック制約（LAW-007）に命中する最小の市場はどこか。
つるはし全体は混雑する（LAW-006）——混雑しない関所の位置を特定する。

# 4. 人材 — 正しいチームか

リポジトリ語彙では：このポジションが要求する **S0 アクセス**（当事者一次情報）と
実行能力を持つのは誰か。CASE-2015 の教訓：見抜くことと殴り切る資本・実行力は別の必要条件。

# 5. 販売 — 届ける方法があるか

リポジトリ語彙では：LAW-001 の第4変換（意思決定への接続）。
構造化された価値が、誰の・どの意思決定に・どの経路で入るのか。

# 6. 永続性 — 10年後も生き残るポジションか

リポジトリ語彙では：**どの不変量（frozen な constraints/）に錨を下ろしているか**。
解凍中の変数の上に建つポジションはラッシュと共に消える（LAW-006 の系）。
反世界（linked_simulations の anti-thesis 分岐）が実現しても生き残る形を書く。

# 7. 隠れた真実 — 他社が気づいていない何を知っているか

リポジトリ語彙では：**地図と領土のズレ**。コンセンサス（世間の地図）と自分の法則・S0観測の差分。
「全員が定数だと思っているが実は変数」または「全員が変わると思っているが変わらない」を1つ特定する。
秘密は必ず S0/S1 の観測（signals/）か採点済みの法則で裏づける——裏づけのない秘密はポエムである。

# 総合判定

7つの答えのうち最も弱いものはどれか。それを補強する次のアクション（観測 or 予測）は何か。
