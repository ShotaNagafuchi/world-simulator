構想の原文：「必要なものを探すとき、AIが偏りなくすべての選択肢を見せてくれる世界を作る」。探す人（複数角度で聞いて網羅→全部見たと確信できる→適合性で並ぶ）と売る人（出るべき質問で出ているか診断→原因URL→直せる）の両輪フライホイール。Google と ChatGPT の間の「検索と網羅性」の空白を取りに行く。

自己申告された課題：PayPal 黎明期のように (a) 明確な優位性がない、(b) ターゲット市場を絞りきれていない、(c) 独占戦略が未定。以下、装置に通した結果。

# 1. エンジニアリング — ブレークスルーか、段階的改善か

変数化を狙うべき定数は「**AI推薦の網羅性は検証不能**」である。SIG-2026-003（S0）が示す通り、現状の AI 推薦は非決定・知名度偏重・母集団非明示。「もっと良い聞き方をするUI」（複数角度質問の自動化）は段階的改善であり、プラットフォームの次のアップデートで消える位置。ブレークスルーは**業種の母集団データ（全ベンダーリスト）× 質問空間の体系的巡回**を掛け合わせて「この推薦は全n社を分母に持つ」と宣言可能にすること——網羅を検証可能な主張に変える。これは検索技術ではなくデータ資産の問題であり、プラットフォームが構造的にやらないこと（下記2）だから防御になる。

# 2. タイミング — なぜ今か

窓を定義した：**プラットフォームが網羅性の責任を引き受けるまで**（PRED-2026-004、引き受けない側に p=0.7 で賭けた）。CON-002 の裏返しが窓を守る——「全部見た」と宣言することは責任の引き受けであり、全カテゴリを扱うプラットフォームには回避誘因がある。ニッチプレイヤーは特定業種でなら分母を保証できる。早すぎ判定：AI 経由の選定行動がまだ主要チャネルでないなら早すぎ（SIG-2026-001 は進行を支持）。遅すぎ判定：PRED-2026-004 が true になったら「地図」の窓は閉じ、「外部監査」（審判の検証）に転進する。

# 3. 独占 — 大きなシェアを取れる小さな市場から始めているか

**現状の最弱点（自己申告 (b)(c) と一致）。**「検索と網羅性の間」は市場ではなく方向である。装置の答え：独占単位は**業種 × 国の監査済み地図**（例：日本の勤怠管理ツール、日本の特定業種向けSaaS）。PayPal の教訓の翻訳——eBay パワーセラーに相当する「痛みが最も鋭い最初のユーザー」は両側で特定できる：
- 売る側：**高品質・低知名度ベンダー**。知名度ソートでは永遠に埋もれ、適合性ソートなら浮上する側。参加誘因が構造的に最強（LAW-002 の沈黙する良品）
- 探す側：**選定を他人に説明する義務がある買い手**（稟議・調達・デューデリ）。CON-001 により人は本当は全部見たくない——網羅の顧客価値は「全部見ること」ではなく「**全部見たと第三者に説明できること**」。日本の稟議文化は説明責任密度が世界最高クラスで、ビーチヘッドとして合理的

# 4. 人材 — 正しいチームか

保有：GPR 実験基盤（S0 データ生成能力）、tkbase 経由の業種接点。欠け：業種の母集団データを持つ主体（業界団体・展示会主催者・既存レビューサイト）との提携獲得能力、および中立性の制度設計（下記5-6）をやり切るガバナンス。CASE-2015 の教訓通り、見抜くことと殴り切ることは別条件。

# 5. 販売 — プロダクトを作るだけでなく、届ける方法があるか

入口は売り手側診断（現行 Sighted、LAW-003 の期待需要が追い風）で現金化し、地図データを蓄積する順序が正しい。**ただしここに LAW-008 の罠が口を開けている**：売り手課金で「偏りのない地図」を建てた者は、格付け機関と同じ均衡に落ちる。設計要件は監査法人型の分離壁——**診断（計測）は売るが、順位は絶対に売らない**。買い手側への配達物は「稟議に添付できる網羅証明レポート」であり、これは買い手課金の可能性を開く（評価が動的に更新され続けるため公共財化しにくい＝LAW-008 の延命設計 (b) に該当）。

# 6. 永続性 — 10年後も生き残るポジションか

錨は CON-002：責任は生成できない。反世界（プラットフォーム吸収、p=0.30）が来ても、**審判は自己監査できない**（LAW-008）——プラットフォームが推薦と広告を融合させるほど、独立検証の需要はむしろ増える。よって生存形態は「よりよい検索」ではなく「AI推薦の会計監査」。最大の脅威は外部ではなく自分の収益モデルである：売り手課金比率が上がるほど、このポジションの存在理由（中立）が内側から溶ける。格付け機関は腐っても儲かったが（CASE-2008）、参入期の無名プレイヤーは腐ったら即死する——寡占の耐腐敗性はまだない。

# 7. 隠れた真実 — 他社が気づいていない何を知っているか

コンセンサスの地図：「AI検索の網羅性が低いのは技術が未熟だから。モデルが良くなれば解決する」。
領土の観測とのズレ：**網羅性の不在は技術問題ではなく、責任問題（CON-002）＋収益構造問題（LAW-008）である。** モデルがいくら賢くなっても、母集団を宣言する責任とバイアスのない順位づけは、プラットフォームの誘因構造から出てこない。だから「網羅の保証」は独立レイヤーに構造的に残り続ける。裏づけ：SIG-2026-003（S0：どのエンジンも母集団を明示しない）＋ CASE-2008-rating-agencies（審判の自己監査が構造的に不可能な歴史）。

# 総合判定

7問中の強い答え：2（タイミング＝責任回避が窓を守る）、7（隠れた真実＝網羅は責任問題）。
最弱の答え：**3（独占の最小市場が未選定）と 5-6（中立と収益の分離設計が未着手）**——自己申告の課題 (b)(c) と装置の診断は一致した。(a) の「明確な優位性がない」については装置の答えは明確で、優位性は検索技術ではなく「業種×国の母集団データ＋S0質問空間データ」の非対称にしかない。
