# 主張

摩擦が桁で下がると（LAW-004）、その工程に張り付いていたロックインも溶解する。**しかしロックインは「次の希少レイヤー」に再形成するのではない。** ロックインは「次のルーティング機能の所在」に再形成する。

価値の移動先（LAW-004）とロックインの移動先は**異なる法則に従う：**

- **価値**は残った摩擦（希少性）に集中する → LAW-004
- **ロックイン**はルーティング機能（A と B を結ぶ関所）に形成される → 本法則

両者は一致することもあるし（Google：ルーティングが信用判定を兼ねた）、乖離することもある（信用・検証レイヤー：価値は集中するがロックインは形成できない）。**乖離するケースを見誤ると、「価値が高いが誰も独占できない」領域を「次の独占が生まれる場所」と誤予測する。**

# ルーティングの定義

ロックインの最小構造は「ルーティング」である：A が B に到達するためにルーターを経由せざるを得ず、B も A に到達するためにルーターを経由せざるを得ない状態。両側が依存することで、どちらも抜けられない。

- Google：ユーザーは Google で検索しないとサイトを見つけられない。サイトは Google に載らないとユーザーに見つけてもらえない
- PayPal：買い手は PayPal がないと売り手に払えない。売り手は PayPal がないと買い手から受け取れない
- Amazon：消費者は Amazon でないと商品を見つけられない。出品者は Amazon でないと消費者に届けられない

**ルーティング機能を持たないレイヤーにはロックインは形成されない。** 価値がいくら集中しても、ルーティングがなければ両面ネットワーク効果が働かず、利用者は自由に乗り換えられる。

# 価値とロックインが乖離するケース（不可能領域）

信用・検証・評価レイヤーは LAW-005 により価値が集中するが、ロックインは構造的に不可能である：

- **プロトコルで検証する場合**：責任の所在がないため信用されない（CON-002 が阻止）
- **主体が検証を引き受ける場合**：その主体は「審判」になり、構造的に腐る（LAW-008 が阻止）

CON-002（human-nature 層、不変）× LAW-008 の組み合わせが「不可能領域」を形成する：**価値は集中するが独占できない**。この領域を予測のターゲットにすると、「価値が高いのに誰も勝てない」という泥沼を予測する結果になる。

# メカニズム

1. 摩擦崩壊：ある工程のコストが桁で下がる
2. 旧ロックイン溶解：その工程のルーティング機能が無力化する
3. 価値の移動（LAW-004）：残った摩擦に需要が殺到 → 価値はここに集中
4. ルーティングの移動（本法則）：新しい「A と B を結ぶ関所」がどこにできるかを探す → ロックインはここに形成
5. **3 と 4 が一致するなら次の独占者が生まれる。乖離するなら、価値はあるが独占者のいない分散市場になる**
6. 旧独占者の構造的不利：旧独占者は (a) 旧資産の減損を認められない、(b) 旧収益モデルを自ら破壊するインセンティブがない、(c) 新ルーティングに必要な能力が旧能力と直交する → 新参者の窓が開く

**新参者が窓を通過する条件**（CON-004 と結合）：
- 新ルーティングの初期に必要な信頼が低い（小さな賭けから始められる）
- フィードバックループが短く、価値が自明（因果証明コストが安い）
- 両面ネットワーク効果がユーザー数に対して超線形に効く

# 予測誤りのパターン（本法則の発見経緯）

**「価値-ロックイン混同」型の誤り：**

AI 時代のロックインの移動先を予測する際、「検証レイヤーに価値が集中する（LAW-005）→ ∴ ロックインも検証レイヤーに形成される」と推論した。しかし検証レイヤーは CON-002 × LAW-008 の不可能領域であり、ロックインは形成できなかった。

Google が検証（PageRank による信用判定）を提供した「ように見えた」ことが混同を助長した。実態は、Google のロックインは検証機能ではなく**ルーティング機能**（注意の配分）の両面ネットワークにあった。検証はルーティングの手段であって、ロックインの源泉ではなかった。

この誤りを避けるための手順：
1. 価値の移動先を LAW-004 で特定する（残った摩擦 → 検証・信用等）
2. **別途**、ルーティングの移動先を特定する（新しい「A × B」の関所はどこか）
3. 1 と 2 が一致するか検証する
4. 一致しない場合、「価値はあるが独占できない領域」と判定する
5. ロックインの予測は 2 から出す。1 からは出さない

# 証拠

- CASE-1998-google-pagerank: Web がコンテンツ配信の摩擦を崩壊させた → 旧ロックイン（物理流通網を持つメディア企業）が溶解 → Google は検証を提供したのではなく、「ユーザー × サイト」のルーティングを獲得した。ロックインはルーティングの両面ネットワーク（ユーザーが来る → 広告主が来る → 資金でサービス改善 → ユーザーがさらに来る）から生まれた
- CASE-1956-containerization: 荷役コストの崩壊 → 港湾労働組合のロックインが溶解 → 新ロックインは「荷主 × 港湾」のルーティング（どの港に寄港するかの選択）を握った Maersk 等の物流ネットワークに形成
- CASE-1999-paypal: インターネットが取引の摩擦を下げた → 物理的支払インフラのロックインが溶解 → 新ロックインは「買い手 × 売り手」のルーティング（決済の橋渡し）を握った PayPal の両面ネットワークに形成

# 反証可能性

新しい摩擦崩壊イベントごとに：(a)「ルーティング機能の所在」と「価値の集中先」を事前に分けて記録し、(b) ロックインが実際にどちらに形成されたかを追う。ルーティングではなく価値集中先にロックインが形成された事例が積み上がれば、本法則の核心（ルーティング条件）が死ぬ。

# 予測への使い方

入力：(1) 崩壊中の摩擦（LAW-004）、(2) 価値の移動先（LAW-004 + LAW-005）、(3) 新しいルーティング機能の候補（新しい「A × B」の関所）
出力：
1. 価値の移動先とルーティングの移動先は一致するか？
2. 一致 → 独占のチャンス。CON-004 で「誰が取れるか」を判定
3. 不一致 → 価値層は分散市場。ロックインはルーティング層にのみ形成
4. 旧独占者の資産が新ルーティングで負債になるか → なるなら新参者の窓

**現在の最重要の問い**：AI が検索・比較・要約の摩擦を崩壊させている。Google の「ユーザー × サイト」ルーティングが溶解中。新しい「A × B」のルーティングは何か？　価値は検証レイヤーに行く（LAW-005）が、ロックインはそこには形成されない。ルーティングの所在を別途探す必要がある。
