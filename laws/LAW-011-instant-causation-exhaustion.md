# 主張

因果が即時（行動の結果が秒-分で確認できる）の必須行動は、2020年代前半までにほぼ全てプラットフォーム化された。

| 行動 | 因果の速さ | プラットフォーム |
|---|---|---|
| 遠くの人に金を送る | 秒 | PayPal / Wise |
| 知りたいことを見つける | 秒 | Google / ChatGPT |
| 今すぐ車に乗る | 分 | Uber |
| 何でも質問して答えを得る | 秒 | ChatGPT |
| メッセージを送る | 秒 | LINE / WhatsApp |
| 目的地への経路を知る | 秒 | Google Maps |
| 異言語のテキスト/会話を理解する | 秒 | DeepL / Google Meet |
| 今すぐ何かを買う | 分 | Amazon |
| 写真/動画を共有する | 秒 | Instagram / TikTok |

残っている未解決の行動は全て「因果が遠い」——比較・判定・選択・診断は、行動の「正しさ」が後（週-年単位）に判明する。CON-004（因果証明コスト）により、因果が遠い領域ではPayPalモデルのロックイン（データ蓄積 → 精度向上 → ユーザー増 → データ蓄積の好循環）が形成されない。

# メカニズム

因果が即時の行動がなぜ先にプラットフォーム化されたか：

1. **CON-004**: フィードバックが速い → ユーザーがすぐ価値を実感 → 急速に普及
2. **CON-006**: 10倍便利が体感しやすい → 信用の壁を超える
3. **CON-013（双曲割引）**: 即時の報酬は過大評価される → 「今すぐ結果が分かる」サービスは採用が速い
4. **CON-011（現状維持バイアス）**: 即時の利得が大きいと現状維持コストを超える
5. **ネットワーク効果の速度**: 因果が即時 → ユーザーが増える速度が速い → 勝者が早く決まる → 窓が早く閉じる

つまり因果が即時の領域は**最も速くプラットフォーム化され、最も速く窓が閉じる**。2026年時点で窓が開いているということは、その行動の因果が遠いことの証左である。

# 次のイノベーションの2つの経路

## 経路A: 既存プラットフォームの10倍効率化

因果が即時の行動は解決済みだが、解決の「やり方」は初期のアーキテクチャに固定されている。全く別のアプローチで同じ行動を10倍効率化できれば、既存プラットフォームを置き換える窓が開く。

歴史的前例：
- Yahoo（ディレクトリ）→ Google（PageRank）: 同じ「検索」を全く別のアーキテクチャで10倍に
- タクシー会社 → Uber: 同じ「配車」を全く別のアーキテクチャで10倍に
- Google検索 → ChatGPT: 同じ「知りたい」を全く別のインターフェースで10倍に

今のプラットフォームも同様に「全く別のやり方」で10倍化される可能性がある。AIはその「別のやり方」の最有力候補。

## 経路B: 因果の遠い領域の因果短縮

因果が遠いからPayPalモデルが成立しない。ならば**因果を短くする技術**を作ればいい。

- 保険比較: 「本当に良かったか」は事故時に分かる → **シミュレーションで「もし事故が起きたらこの保険はこう支払う」を即座に可視化** → 因果を擬似的に短縮
- 契約書: 「不利だったか」はトラブル時に分かる → **過去の判例データベースで「この条項でトラブルになった確率X%」を即座に表示** → 因果を統計的に短縮
- 医師選び: 「良い医師だったか」は治療後に分かる → **治療結果データベースで「この医師のこの手術の成功率X%」を即座に表示** → 因果を統計的に短縮

因果短縮が成功すると、その領域がCON-004の閾値を超え、PayPalモデルが新たに成立する可能性がある。

# 反証可能性

- 因果が即時で未プラットフォーム化の必須行動を探す。見つかればこの法則の「全て」が過大主張
- 因果が遠い領域でPayPalモデルが成立する事例を探す。見つかればCON-004の適用範囲が狭い

# 予測への使い方

入力：(1) 対象行動の因果の速さ、(2) 既存プラットフォームの有無
出力：
- 因果即時 × 未プラットフォーム化 → **本当にないか再確認**（見落としの可能性が高い）
- 因果即時 × プラットフォーム化済み → **経路A: 全く別のやり方で10倍化できるか？**
- 因果遠い × 未プラットフォーム化 → **経路B: 因果を短縮する技術で閾値を超えられるか？**
