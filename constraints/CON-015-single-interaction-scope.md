# 制約の内容

スタートアップが解決できるのは、**1つのインタラクション（1人のユーザー × 1つのサービス/相手）の中の摩擦**だけである。複数の組織・プラットフォームにまたがる横断的課題は、以下の理由で構造的に解決不能：

1. **調整コスト**: N個の組織の協力が必要 → 調整コストはO(N²)で増大
2. **インセンティブの不一致**: 各組織は自社最適で動く（CON-009）。全体最適のためにコストを負う理由がない
3. **データの囲い込み**: 各組織のデータは競争優位の源泉（LAW-012）。共有するインセンティブがない
4. **誰も所有しない問題**: 横断的課題は「誰の問題でもない」ため、解決に予算がつかない

# 成功したイノベーションの共通点

PayPal、Google、Uber、ChatGPT——全て**1つのインタラクションの摩擦**を解決した：

| イノベーション | 解決したインタラクション | 横断的ではなかった理由 |
|---|---|---|
| PayPal | 送金者→受取人（1対1） | 銀行間の連携は不要。PayPalが間に入るだけ |
| Google | 検索者→Webページ（1対多だがクロール） | Webサイトの協力は不要。勝手にクロール |
| Uber | 乗客→ドライバー（1対1） | タクシー会社の協力は不要。ドライバー個人が参加 |
| ChatGPT | 質問者→知識（1対1） | データ提供者の協力は不要。訓練データで完結 |

**全てに共通: 既存のプレイヤーの「協力」を必要としない。** 既存組織を迂回するか、個人レベルで参加させた。

# 判定基準

「この問題を解決するのに、何社の協力が必要か？」
- 0社（自分だけで完結）→ 最も有望
- 1社（相手方1社）→ 有望（相手が得するなら）
- 2社以上 → 構造的に困難
- 業界全体 → 不可能（規制でしか解決しない）

# 解凍条件の観測方法

- Open Banking/Open Health等のAPI標準の普及率
- AIエージェントが「個人の代理」として各組織と個別にやり取りし、組織間協力なしに統合を実現する事例

# 依拠している法則・ポジション

LAW-012（統合のパラドックス）の上位制約。CON-009（インセンティブ超反応）の組織版。
