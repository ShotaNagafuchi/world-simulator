# 制約の内容

人間が情報を信用するのは、それが間違っていたときに誰かが代償を払う構造（skin in the game）があるときである。署名、実名、資格、免許、監査、保証——信用の制度はすべて「責任の所在の固定」の変奏である。生成コストがいくら下がっても、責任を引き受けるコストは技術では下がらない。

# 考古学（物理層までの遡行）

- human-nature 層：信用とは相手の将来行動の予測モデルであり、裏切りへの罰の存在が予測を安定させる。これは進化的に固定された協力の条件（互恵性）で、技術速度では動かない
- institution 層はこの本性の凝固物にすぎない：医師免許も監査法人も「間違えたら失うものがある者」の制度化
- **AI が下げない摩擦（LAW-004）の正体はこれである。** 文章は生成できても、責任は生成できない

# 解凍条件の観測方法

- 高リスク領域での匿名/AI情報の採用率（例：判例引用、診療ガイドライン、与信判断に責任主体なし情報が使われるか）
- 責任代替メカニズムの実務浸透（保険つきAI回答、ステーク型の情報市場が主要産業で使われるか）

# 依拠している法則・ポジション

LAW-002（発信の期待リターンに責任リスクが入る）と LAW-005（信用レイヤーの価値）の両方がこの制約に依拠する。この制約が解凍したら、「信用ある一次情報の構造化」というポジション自体の前提が崩れる——**このリポジトリの適用事業にとって最重要の不変量**であり、定点監視の対象。
