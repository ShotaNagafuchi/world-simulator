# world-simulator

**過去に世界を変えた者たちから法則を抽出し、将来の世界をシミュレーションし、予測を回して法則を淘汰する台帳。**

これはナレッジベースではない。予測装置である。
情報を溜めるのではなく、「法則 → 予測 → 採点 → 法則の淘汰」というループを回すために存在する。

ポジショニング：**事業の型集（digital entrepreneur 集）ではなく、未知領域の探索装置。**
法則（地図）と世界（領土）のズレを、予測という賭けで計測し続ける。地図と領土のズレ＝未知（unknowns）を、賭けの形で既知に変換していく装置である。unknown unknowns の多い領域（エネルギー、宇宙、Web3、EV）ほど、この装置の限界効用は高い。

## 第一原理

このリポジトリは次の観察から出発している：

- 情報の希少性と市場価値は一致しない。希少な一次情報も、発信・構造化・信用・意思決定接続の4変換を通らなければ価値化されない
- 本当に貴重な発信源ほど、自発的には発信しない
- 成長市場では、確定した成果ではなく「期待」に資金が流れる
- 技術は価値構造を直接変えない。特定の摩擦を下げることで資金の流路を変える
- 生成コストが下がるほど、信用・検証・選別レイヤーの相対価値が上がる

だから個別サービスや「伸びている市場」を追うのではなく、**資金流動の法則 × 摩擦低下の観測**から未来を構成する。コンテンツ発信・AEOはその最初の適用ドメインにすぎない。エネルギー、Web3、宇宙、EV——「明確な未来像があり、まだ誰も信じていない」領域すべてに同じエンジンを適用する。

## 構造

```
laws/          法則 — 反証条件つきの原則。予測の成績によって淘汰される
cases/         事例 — 過去に世界を変えた出来事。法則の証拠として法則にリンクする
history/       過去 — 不変のメモリ。年表・コスト系列など、法則と事例の原料（腐らない）
state/         現在 — ドメインごとの現在地スナップショット。AI/スクレイピングで定期更新し、古くなると警告される
signals/       変化 — 出典・信用クラス・有効期限つきの観測。予測に接続されなければ腐って捨てられる
predictions/   予測 — 確率・期限・解決基準つきの賭け。open/ → resolved/ に移動して採点される
simulations/   世界線 — 法則を組み合わせたシナリオ木。反世界（anti-thesis）と決定的観測のリストが必須
engine/        エンジン — スキーマ検証と採点（Brierスコア → 法則への成績伝播）
scoreboard/    成績表 — 自動生成。法則ごとの予測成績
```

時間の三層：**過去（history = 不変）／現在（state = 定点更新）／変化（signals = 期限つき）**。
それぞれ腐り方が違うので、置き場所と更新規則を分ける。

## 運用ループ

```
① Observe   state/ を更新する（WebSearch/スクレイピング）。何を見るかは世界線の「決定的観測」リストが指定する
② Extract   state の差分のうち予測を動かすものを signals/ に切り出す（出典・信用クラス・期限つき）
③ Connect   シグナルを法則に接続する。接続できないシグナルは追加しない
④ Predict   法則から反証可能な予測を生成する（確率・期限・解決基準つき）
⑤ Resolve   期限が来た予測を解決し resolved/ へ移す
⑥ Propagate `make score` — Brierスコアを計算し、引用された法則へ成績を伝播する
⑦ Revise    成績の悪い法則の confidence を下げ、kill condition に達した法則を falsified にし、世界線の分岐確率を更新する
```

## 破壊的メカニズム（ベストプラクティスに反する意図的な設計）

1. **反証駆動** — 法則には `kill_conditions` が必須。殺し方の書けない法則は登録できない（LAW-000 メタ法則）
2. **成績の遡及（blame propagation）** — 予測が外れると、その予測が引用した全法則の成績が下がる。法則は市場淘汰される
3. **シグナルの腐敗** — 全シグナルに `expires` が必須。期限切れで予測に未接続のシグナルは削除する。「情報を溜め込むこと」自体を禁止する
4. **反世界の義務化** — 全シミュレーションに「テーゼが死ぬ世界線」を確率つきで含める。自己確証バイアスを構造で殺す
5. **信用の重み付け** — シグナルは出典の信用クラス（S0 当事者一次 / S1 第三者一次 / S2 二次 / S3 AI生成）を持つ。S3 は原則入れない。このリポジトリ自体が「信用を伴った一次情報が価値を持つ」というテーゼを実装する
6. **決定的観測（Value of Information）** — 全世界線に「答えが分岐確率を最も動かす観測」のランクリストを必須化。観測は網羅ではなく、情報価値の高い順に行う。state/ の定点更新はこのリストを見て回る
7. **現在地の鮮度強制** — state/ には更新周期（`refresh_every_days`）が必須。周期を過ぎた現在地スナップショットは validate が STALE 警告を出す。古い地図で賭けることを構造的に防ぐ

## 使い方

```bash
make validate   # 全ファイルのスキーマ・参照・期限を検証
make score      # 予測を採点し scoreboard/SCOREBOARD.md を再生成
make test       # エンジンのテスト
```

## ID 規約

| 種別 | 形式 | 例 |
|------|------|-----|
| 法則 | `LAW-NNN` | `LAW-004` |
| 事例 | `CASE-年-名前` | `CASE-1956-containerization` |
| 歴史 | `HIST-NNN-名前` | `HIST-001-information-gatekeepers` |
| 現在地 | `STATE-ドメイン` | `STATE-content` |
| 観測 | `SIG-YYYY-NNN` | `SIG-2026-001` |
| 予測 | `PRED-YYYY-NNN` | `PRED-2026-002` |
| 世界線 | `WORLD-YYYY-NNN` | `WORLD-2026-001` |
